# app-v2

`app-v2` - це greenfield-перезапуск застосунку з нуля.

Старий проєкт залишаємо як робочий інструмент для зарплати, виплат і замовлень. Ця папка створена для нового продукту, де весь наявний функціонал буде збережений, але вже на нормальній модульній структурі без історичного хаосу.

## Ціль нового застосунку

- зберегти весь поточний функціонал;
- відокремити персональний облік від операційного бізнес-застосунку;
- побудувати чисту структуру модулів, ролей і процесів;
- перенести планування, Gantt, платежі, штрафи, файли, лог дій і адміністрування в єдину систему;
- одразу закласти новий сучасний дизайн.

## Модулі, які мають бути в новому продукті

1. Дашборд та аналітика
2. Реєстр замовлень
3. Детальна картка замовлення
4. Планування виробництва та Gantt
5. Платежі та рознесення оплат
6. Провини / відрахування
7. Команда, ролі та права доступу
8. Файли по замовленнях
9. Історія дій
10. Налаштування, резервні копії, відновлення

## Структура стартового каркаса

```text
app-v2/
  src/
    app/         # маршрути, конфігурація, точка входу застосунку
    layout/      # оболонка інтерфейсу: sidebar, topbar, page shell
    modules/     # окремі бізнес-модулі
    shared/      # mock-дані, універсальні конфіги, спільні утиліти
    styles/      # нова візуальна система
```

## Поточний статус

- стартовий фронтенд-каркас створено;
- нова навігація і дизайн-система закладені;
- окремі сторінки під усі ключові модулі вже розведені;
- логіку та API будемо переносити поетапно, модуль за модулем.

## Наступна черга

1. Підключити авторизацію та ролі
2. Перенести реєстр замовлень
3. Перенести картку замовлення
4. Перенести фінанси, платежі та штрафи
5. Підключити реальний Gantt і планування
