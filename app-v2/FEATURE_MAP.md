# Feature Map

Нижче зафіксовано, який функціонал уже є в старому застосунку і куди він переходить у новому `app-v2`.

## 1. Авторизація та ролі

- Login / token auth
- Поточний користувач
- Ролі: `super_admin`, `admin`, `manager`, `constructor`
- Видимість для менеджера по окремих блоках

Цільовий модуль:
- `auth`
- `access-control`

## 2. Дашборд

- Борг конструкторам
- Борг менеджерам
- Нерозподілені кошти
- Зведення по конструкторах
- Зведення по менеджерах

Цільовий модуль:
- `dashboard`
- `analytics`

## 3. Замовлення

- Реєстр
- Пошук
- Фільтри
- Архів
- Календар
- Gantt
- Детальна картка замовлення
- Редагування замовлення
- Видалення замовлення

Цільовий модуль:
- `orders`
- `order-detail`
- `planning`

## 4. Виробничі етапи

- Передано в роботу
- Дедлайн конструктиву
- План монтажу
- Факт монтажу
- Тривалість конструктиву
- Тривалість комплектації
- Тривалість предзбірки
- Тривалість монтажу

Цільовий модуль:
- `planning`
- `gantt`
- `production-flow`

## 5. Платежі

- Додавання платежу
- Автоматичне рознесення
- Ручне рознесення
- Історія платежів
- Видимість платежів для ролей
- Видалення платежу

Цільовий модуль:
- `payments`
- `payment-allocation`

## 6. Виплати та зарплата

- Розрахунок бонусу конструктора
- Розрахунок бонусу менеджера
- Виплата по стадіях
- Індивідуальні відсотки
- Fixed bonus
- Відображення боргу / залишку

Цільовий модуль:
- `salary`
- `settlements`

## 7. Провини / відрахування

- Створення
- Список
- Оновлення
- Видалення
- Погашення

Цільовий модуль:
- `deductions`

## 8. Файли

- Список файлів по замовленню
- Посилання
- Завантаження файлів
- Структура папок
- Видалення

Цільовий модуль:
- `files`
- `documents`

## 9. Користувачі

- Створення користувачів
- Редагування
- Видалення
- Налаштування ролі
- Налаштування зарплатної моделі
- Налаштування видимості для менеджера

Цільовий модуль:
- `users`
- `roles`

## 10. Системні можливості

- Історія дій
- Налаштування
- Backup
- Restore
- Fix schema / service tools
- Очистка БД

Цільовий модуль:
- `activity-log`
- `settings`
- `system-tools`

## Принцип перенесення

Переносимо не "по файлах старого UI", а по доменах:

1. Auth + roles
2. Orders
3. Order detail
4. Planning / Gantt
5. Payments / salary / deductions
6. Files
7. Analytics
8. System tools
